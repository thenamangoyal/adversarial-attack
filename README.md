# Binary Iterative Method for Non-targeted Adversarial Attack

**Naman Goyal, Milan Chaudhari** (Indian Institute of Technology, Ropar)

[![Paper](https://img.shields.io/badge/paper-PDF-b31b1b)](Binary_Iterative_Method_Adversarial_Robustness.pdf)

We propose the **Binary Iterative Method (BinIM)**, a non-targeted white-box adversarial attack. BinIM builds on the Basic Iterative Method but replaces the fixed epsilon step with a binary search over the step size (halved each iteration), with multi-round restarts returning the best local optimum. Fixed epsilon-ball search is O(n) and not guaranteed to land near a local optimum; binary search is O(log n) and settles near one. On 1000 images sampled from ImageNet, BinIM drives the classifier to misclassify with confidence up to 0.995 while reducing the probability of the true label to about 2.2e-09.

Originally submitted to AAAI 2021 (not published). Released on arXiv in 2026 as-is, with the supplementary material merged as an appendix.

## Results

Attack generated with gradients from Inception V3; evaluated on three pre-trained classifiers. Numbers are post-attack classification accuracy (lower = stronger attack).

| Model | Clean | FGSM | BIM | VAM | **BinIM (ours)** |
|---|---|---|---|---|---|
| Inception V3 | 0.958 | 0.356 | 0.0252 | 0.628 | **0.009** |
| Inception V2 | 0.855 | 0.882 | 0.839 | - | **0.683** |
| ResNet V2 152 | 0.906 | 0.835 | 0.828 | - | **0.668** |

FGSM = Fast Gradient Sign Method, BIM = Basic Iterative Method, VAM = Virtual Adversarial Method.

## Code

The code in `code/` is the original 2017 implementation, released as a historical artifact (TensorFlow 1.x, cleverhans).

Large assets are attached to the [v1.0 GitHub Release](https://github.com/thenamangoyal/adversarial-attack/releases/tag/v1.0):

- `ckpts.zip` (~800 MB): pre-trained TF-slim checkpoints (inception_v3, inception_v2, resnet_v2_152)
- `dataset.zip` (~260 MB): the 1000 sampled ImageNet images plus the adversarial images generated by each method (FGM, BIM, VAM, BinIM)

Setup and usage: see [code/README.md](code/README.md).
